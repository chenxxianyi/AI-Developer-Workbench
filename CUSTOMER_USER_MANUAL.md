# AI Developer Workbench 客户使用手册

## 1. 产品简介

AI Developer Workbench 是一款面向开发团队和独立开发者的 AI 开发质量工作台。用户可以通过网页上传截图、代码、项目 ZIP 或数据库结构内容，快速生成质量分析报告和改进建议。

当前系统提供 5 个工具：

| 工具 | 用途 |
| --- | --- |
| UI 质量审查 | 根据页面截图或前端代码分析 UI/UX 问题 |
| 项目诊断 | 上传项目 ZIP，检查项目结构、依赖、工程质量和风险 |
| Agent 配置生成 | 生成 AGENTS.md、TASK_PLAN.md 等 AI Coding 配置文件 |
| API 文档生成 | 根据代码、项目文件或接口描述生成 API 文档 |
| 数据库结构审查 | 分析 SQL、GORM、Prisma 等数据库结构并给出优化建议 |

## 2. 访问系统

如果使用 Docker 部署，启动后在浏览器打开：

```text
http://localhost:5173
```

后端接口默认地址：

```text
http://localhost:8080/api
```

如果由管理员部署到服务器，请使用管理员提供的网址访问。

## 3. 页面说明

### 首页

展示产品介绍和入口。点击进入工作台后，可以选择不同分析工具。

### 工作台

工作台包含：

- 总报告数
- 平均评分
- 工具使用情况
- 最近报告
- 五个分析工具入口

点击任意工具卡片即可开始使用对应功能。

### 系统状态

系统状态页面用于查看：

- 服务是否正常
- 当前 AI 服务商
- 文本模型
- 视觉模型
- 图片和 ZIP 上传大小限制

如果系统显示“服务异常”，请联系管理员检查后端服务、数据库或 AI 配置。

## 4. 工具使用说明

### 4.1 UI 质量审查

适用于审查页面设计、视觉一致性和可访问性问题。

使用步骤：

1. 进入“UI 质量审查”。
2. 输入报告标题。
3. 选择分析模式：
   - 截图：只上传页面截图。
   - 代码：只粘贴前端代码。
   - 两者：同时提供截图和代码，分析更完整。
4. 上传截图或粘贴代码。
5. 可选填写页面类型、目标风格和补充说明。
6. 点击“开始分析”。
7. 在右侧查看评分、问题和改进建议。

截图支持：

- PNG
- JPG/JPEG
- WebP
- 最大 20MB

小提示：截图上传区域支持点击选择文件，也支持截图后点击上传框并按 `Ctrl + V` 粘贴。

### 4.2 项目诊断

适用于检查项目整体健康度、工程结构和潜在风险。

使用步骤：

1. 进入“项目诊断”。
2. 输入报告标题。
3. 可选填写项目名称。
4. 上传项目 ZIP 文件。
5. 可选填写技术栈，例如 `Vue 3 + Go + MySQL`。
6. 选择分析深度：
   - 基础：快速扫描关键风险。
   - 标准：平衡速度与覆盖范围。
   - 深度：更全面的工程审查。
7. 可选填写项目描述。
8. 点击“开始诊断”。
9. 查看健康评分、问题列表和改进建议。

ZIP 上传限制：

- 文件格式：`.zip`
- 默认最大 20MB
- 系统只进行静态分析，不会执行上传项目中的代码。

### 4.3 Agent 配置生成

适用于为 AI Coding 工具生成项目协作说明和任务计划文件。

使用步骤：

1. 进入“Agent 配置生成”。
2. 输入标题和项目相关信息。
3. 填写项目技术栈、目标和约束。
4. 点击生成。
5. 查看生成内容。
6. 下载或复制生成的配置文件。

常见输出文件：

- `AGENTS.md`
- `TASK_PLAN.md`
- 其他项目协作说明文件

### 4.4 API 文档生成

适用于根据代码、项目 ZIP 或手动描述生成接口文档。

使用步骤：

1. 进入“API 文档生成”。
2. 输入报告标题。
3. 选择输入方式：
   - 粘贴代码
   - 上传项目 ZIP
   - 手动输入接口说明
4. 填写或上传对应内容。
5. 点击开始生成。
6. 查看生成的 API 文档。
7. 下载 Markdown 或 OpenAPI 相关文件。

### 4.5 数据库结构审查

适用于检查数据库表结构、索引、性能和安全风险。

使用步骤：

1. 进入“数据库结构审查”。
2. 输入报告标题。
3. 粘贴数据库结构内容。
4. 内容可以是：
   - SQL DDL
   - GORM Model
   - Prisma Schema
   - 其他数据库结构描述
5. 点击开始审查。
6. 查看结构评分、问题和优化建议。

注意：系统只分析文本内容，不会连接或修改真实数据库。

## 5. 查看和导出报告

分析完成后，系统会生成报告。报告通常包含：

- 总体评分
- 等级
- 摘要
- 评分详情
- 问题列表
- 改进建议
- 生成文件

在报告页面或结果区域中，可以：

- 查看分析结果
- 下载生成文件
- 导出 Markdown 报告

说明：当前部分报告列表和详情页面可能仍处于逐步完善阶段，请以实际页面显示为准。

## 6. 文件上传要求

| 类型 | 支持格式 | 默认限制 |
| --- | --- | --- |
| 截图 | PNG、JPG、WebP | 最大 20MB |
| 项目文件 | ZIP | 最大 20MB |

建议：

- 上传前删除无关大文件，例如 `node_modules`、`dist`、`.git`。
- 不要上传包含敏感密钥、账号密码、生产数据的文件。
- 如需分析敏感项目，请先脱敏。

## 7. 常见问题

### 7.1 页面打不开怎么办？

请确认：

1. 前端服务是否启动。
2. 访问地址是否正确。
3. Docker 容器是否正常运行。

Docker 部署时可让管理员检查：

```bash
docker compose ps
```

### 7.2 显示“服务异常”怎么办？

可能原因：

- 后端服务未启动。
- MySQL 未启动或连接失败。
- AI 服务配置异常。

请联系管理员处理。

### 7.3 上传失败怎么办？

请检查：

- 文件格式是否正确。
- 文件是否超过大小限制。
- ZIP 文件是否损坏。
- 文件名是否包含特殊字符。

### 7.4 AI 分析失败怎么办？

系统只保存真实 AI 返回的有效结果。若 AI 服务不可用或返回内容无法解析，本次报告会标记为失败，不会用示例内容替代。请联系管理员检查 AI 服务配置后重试。

### 7.5 系统会执行我上传的代码吗？

不会。项目诊断和文档生成只做静态读取和分析，不会运行上传项目中的脚本、二进制文件或构建命令。

## 8. 管理员快速启动

如果使用 Docker：

```bash
docker compose up -d
```

启动后访问：

```text
http://localhost:5173
```

停止服务：

```bash
docker compose down
```

如果需要清空数据库和上传文件，请先确认数据可以删除，再由管理员处理 Docker volumes。

## 9. 使用建议

- 每次分析请填写清晰的标题，方便后续查找。
- 上传项目 ZIP 前建议移除无关依赖目录。
- UI 审查建议同时提供截图和代码，结果更准确。
- 数据库审查建议提供完整建表语句和索引信息。
- 不要上传未脱敏的密钥、Token、账号密码和真实客户数据。
