# UI 审查报告

## 评分

- **视觉层级**: 82/100 - 页面采用左右双栏结构，信息分组明确，标题、表单区、结果区的层级较清晰。主 CTA“开始诊断”具备较强识别度，但次级信息如帮助说明、上传约束、错误提示与结果详情之间的视觉区分仍可进一步拉开，结果区在信息变多后可能出现层级压缩。
- **一致性**: 84/100 - 整体使用统一的圆角、边框、间距和配色 token，组件风格较一致。输入框、按钮、卡片都遵循相似模式，但上传区域、分析深度切换按钮、结果项列表在交互反馈和状态表达上存在轻微不统一，语义也有混用现象。
- **可访问性**: 58/100 - 存在基础标签文本，但多个控件缺少更完整的可访问性支持，例如 label 未显式绑定 input、上传区域主要依赖点击容器触发、按钮切换组未体现 ARIA 状态、焦点样式只有边框变化且不够突出。键盘用户和辅助技术用户的使用体验仍有明显提升空间。
- **色彩与对比度**: 72/100 - 整体配色较现代简洁，状态色定义明确，但 `text-text-secondary`、`text-text-muted` 与浅色背景组合可能存在对比度不足风险，尤其是说明文字、占位符和空状态提示。依赖颜色表达状态的方式也偏多，需要增加文字或图标辅助。
- **响应式设计**: 76/100 - 使用 `grid-cols-1 lg:grid-cols-2` 具备基本响应式能力，桌面端布局合理。但分析深度按钮、结果评分行、问题列表标题与严重级别在小屏上可能拥挤，上传文件名也可能溢出。移动端的触控尺寸、换行策略和结果区滚动体验还需要进一步优化。

## 发现的问题

- **[high] 表单标签未与控件建立显式关联** (可访问性): 当前 `label` 仅作为视觉文本存在，未通过 `for` 与具体 `input` 或 `textarea` 建立关联，屏幕阅读器无法稳定识别字段名称，点击标签也不能稳定聚焦对应控件。
- **[high] 上传区域键盘可用性不足** (可访问性): 上传区域通过可点击 `div` 配合隐藏 `input` 实现，默认不具备良好的键盘交互能力，也缺少拖拽状态、聚焦状态和操作说明，键盘用户难以发现和使用。
- **[medium] 分析深度选项缺少语义化单选结构** (一致性): “基础 / 标准 / 深度”当前由多个普通按钮实现，视觉上像分段选择器，但语义上没有体现互斥选择，也没有选中状态的 ARIA 标识。
- **[medium] 焦点状态不够明显** (可访问性): 输入框当前主要依赖 `focus:border-accent`，缺少清晰的外轮廓或阴影，在复杂背景或低对比条件下不容易被识别。
- **[medium] 说明文字与占位符对比度可能不足** (色彩与对比度): 空状态、上传说明、次级文本和占位符大量使用 `text-text-muted` 或 `text-text-secondary`，若背景较浅，可能无法满足 WCAG 对比度要求。
- **[medium] 状态表达过度依赖颜色** (色彩与对比度): 评分高低、问题严重程度、健康等级主要依赖不同颜色区分，对于色觉缺陷用户不够友好，且英文严重级别 `high/medium/low` 直接展示也不够本地化。
- **[medium] 结果区长内容缺少更强的信息组织** (视觉层级): 诊断结果包含摘要、评分详情、问题列表等内容，随着数据增多，当前以连续卡片堆叠展示会导致扫描成本上升，重点信息不够突出。
- **[low] 移动端横向拥挤风险较高** (响应式设计): 分析深度按钮横向排列、文件名展示和评分行左右分布在小屏设备上容易出现拥挤、换行不美观或内容溢出。
- **[low] 错误提示位置单一且缺少字段级反馈** (一致性): 当前错误主要集中在通用错误框中显示，用户提交失败后需要自行判断具体哪个字段有问题，不利于快速修正。

## 改进建议

- 为所有表单字段补充显式 `label-for` 绑定，并增加 `required`、`aria-describedby` 与字段级错误提示。
- 将文件上传区域改为语义化上传组件，支持键盘操作、拖拽状态、聚焦反馈和上传失败提示。
- 把“分析深度”重构为可访问的单选组或 segmented control，补充选中态与说明文案。
- 统一所有可交互元素的 `hover`、`active`、`focus-visible`、`disabled` 状态，建立一致的交互规范。
- 提升次级文字、占位符、空状态说明与浅色背景之间的对比度，确保满足 WCAG 基本要求。
- 在评分和问题严重级别中加入中文 badge、图标和文字说明，不仅依赖颜色区分信息。
- 优化结果区层级，优先突出总分、等级、核心摘要和最严重问题，其余内容分组展开。
- 针对移动端增加断点优化：按钮换行、文件名省略、列表纵向排列和更大的触控热区。
- 在提交失败或校验失败时自动滚动并聚焦到首个错误字段，减少用户修正成本。
- 为导出、下载、返回等次级操作建立统一按钮样式等级，避免主次操作视觉竞争。
