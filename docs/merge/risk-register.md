# M0 风险登记表

> 编制日期：2026-07-13  
> 基于两个项目的基线分析

## 风险清单

### 1. 认证体系迁移

| 属性 | 值 |
|------|-----|
| **风险等级** | 🔴 高 |
| **触发条件** | JWT 模块从 Builder 迁入 Workbench 时，与 Workbench 现有无认证模式冲突 |
| **影响** | 所有 API 需要加 JWT 中间件，前端所有请求需要加 Authorization 头 |
| **缓解措施** | 先建认证中间件 + Mock 模式兼容；逐步启用权限校验，保留降级开关 |
| **回滚方式** | 移除 JWT 中间件，恢复无认证模式 |

### 2. ID 类型不一致

| 属性 | 值 |
|------|-----|
| **风险等级** | 🟡 中 |
| **触发条件** | Workbench 使用 string UUID，Builder 使用 uint 自增 ID |
| **影响** | API 参数解析、数据库外键关联、前端路由参数类型错误 |
| **缓解措施** | 统一使用 string UUID；旧 Builder 数据迁移时建立 uint→UUID 映射 |
| **回滚方式** | 保留旧 ID 列作为 legacy_id，确保数据可追溯 |

### 3. Tailwind CSS 版本冲突

| 属性 | 值 |
|------|-----|
| **风险等级** | 🟡 中 |
| **触发条件** | Builder 页面使用 Tailwind v3 类名和配置（如 `tailwind.config.ts`），迁入 Workbench v4 环境 |
| **影响** | 样式断裂、颜色丢失、组件变形 |
| **缓解措施** | 逐页迁移时更新类名为 v4 语法；升级 `tailwind.config.ts` → CSS-based config |
| **回滚方式** | 保留原 Builder 样式文件作为参考，必要时回退页面组件 |

### 4. SSE 连接稳定性

| 属性 | 值 |
|------|-----|
| **风险等级** | 🔴 高 |
| **触发条件** | Builder SSE Broker 迁入 Workbench 时，需处理跨域、鉴权、Nginx 缓冲 |
| **影响** | 生成进度无法实时推送，前端需要轮询降级 |
| **缓解措施** | 独立的 SSE 测试套件；Nginx 配置 `proxy_buffering off`；心跳 + 退避重连 |
| **回滚方式** | 保留 Task 轮询 API 作为 SSE 降级方案 |

### 5. 构建沙箱安全

| 属性 | 值 |
|------|-----|
| **风险等级** | 🔴 高 |
| **触发条件** | Builder 构建服务在服务端执行 `npm install && npm run build` |
| **影响** | 恶意 package.json 可能执行任意代码、消耗资源、泄露文件 |
| **缓解措施** | 构建超时限制、输出大小限制、进程隔离（子进程）、文件白名单 |
| **回滚方式** | 禁用构建功能，仅提供预览静态文件 |

### 6. 数据迁移完整性

| 属性 | 值 |
|------|-----|
| **风险等级** | 🟡 中 |
| **触发条件** | 两个项目的 users、projects、tasks 表有字段差异和 ID 类型冲突 |
| **影响** | 历史数据丢失、关联断裂、用户无法登录 |
| **缓解措施** | 编写迁移脚本 + 校验工具 + 回滚脚本；先在测试环境演练 |
| **回滚方式** | 保留旧数据库只读副本，支持按需恢复 |

### 7. 前端路由模式统一

| 属性 | 值 |
|------|-----|
| **风险等级** | 🟢 低 |
| **触发条件** | Builder 使用 `createWebHashHistory`，Workbench 使用 `createWebHistory` |
| **影响** | 路由路径变化（`#/` 前缀消失），书签和分享链接失效 |
| **缓解措施** | Nginx 配置 SPA History Fallback；添加旧 Hash 路由重定向 |
| **回滚方式** | 切换回 Hash 模式，但会影响 SEO 和 URL 美观性 |

### 8. 图标库替换

| 属性 | 值 |
|------|-----|
| **风险等级** | 🟢 低 |
| **触发条件** | Builder 使用 `lucide-vue-next`，统一为 `@lucide/vue` |
| **影响** | 图标名称和导入路径可能不同 |
| **缓解措施** | 建立图标名称映射表，逐页替换导入语句 |
| **回滚方式** | 同时保留两个图标库（不推荐，会增加包体积） |

### 9. 环境依赖

| 属性 | 值 |
|------|-----|
| **风险等级** | 🟡 中 |
| **触发条件** | Builder 需要 MySQL + Redis，Workbench 仅需 SQLite |
| **影响** | 开发环境搭建复杂度增加，CI/CD 需要额外服务 |
| **缓解措施** | 开发环境保持 SQLite 兼容（通过 GORM 驱动切换）；Docker Compose 统一环境 |
| **回滚方式** | 切换数据库驱动配置 |

### 10. 路径穿越安全

| 属性 | 值 |
|------|-----|
| **风险等级** | 🔴 高 |
| **触发条件** | 文件浏览 API 接收用户输入的路径参数，可能构造 `../` 穿越 |
| **影响** | 读取服务器任意文件，泄露密钥和源码 |
| **缓解措施** | 路径归一化 + 前缀校验 + 白名单目录 |
| **回滚方式** | 禁用文件浏览 API，仅允许导出 |

---

## 风险优先级汇总

| 优先级 | 数量 | 风险项 |
|--------|------|--------|
| 🔴 高 | 4 | 认证迁移、SSE 稳定性、构建沙箱安全、路径穿越 |
| 🟡 中 | 4 | ID 类型、Tailwind 版本、数据迁移、环境依赖 |
| 🟢 低 | 2 | 路由模式、图标库替换 |

---

## 缓解优先级

1. **构建沙箱** — 第一时间实现进程隔离和超时
2. **路径穿越** — 文件 API 上线前完成安全校验
3. **认证迁移** — 先建中间件再逐步启用
4. **SSE 稳定性** — 独立测试套件 + Nginx 配置
5. 其他中低风险项在对应阶段处理
