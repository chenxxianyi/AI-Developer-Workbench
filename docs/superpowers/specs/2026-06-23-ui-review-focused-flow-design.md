# UI Review 聚焦任务流优化设计

## 背景

`UI_REVIEW_REPORT.md` 指出 UI Review 页面存在主 CTA 不够突出、标题/返回入口重复、空状态单薄、焦点态与浅灰对比不足、模式选择缺少说明、移动端适配风险较高等问题。用户已确认采用方案 A：聚焦任务流。

## 目标

在不重做全局主题、不引入分步向导的前提下，优化 UI Review 页面首屏任务流，让用户更清楚地完成“填写输入 → 上传素材 → 开始分析 → 查看结果”。

## 范围

仅优化 `frontend/src/pages/tools/UIReviewPage.vue`：

- 页面头部只保留一个主要 H1，去掉页内“返回工作台”按钮，避免与全局 Header 重复。
- 顶部增加任务说明和轻量流程提示。
- 输入卡片强化结构：标题从“输入参数”调整为“1. 输入与素材”。
- 模式选择增加说明文案，并在按钮内展示每种模式适用场景。
- 上传区继续支持点击、拖拽文案和 `Ctrl+V` 粘贴，增强边框与文字对比。
- 结果区空状态从一句提示升级为结果预览，说明将输出评分、问题、建议和文件。
- 主 CTA 放在表单底部右侧，按钮尺寸、权重和禁用态说明更明确。
- 保持现有 `grid-cols-1 lg:grid-cols-2` 响应式结构，移动端自然单栏。

## 非目标

- 不改全局导航、Sidebar 状态卡片或 Header。
- 不新增深色主题。
- 不改后端接口和报告数据结构。
- 不引入新的组件库。

## 测试策略

扩展 `frontend/src/pages/tools/UIReviewPage.test.ts`：

- 验证页面展示单一聚焦 H1 和任务流说明。
- 验证页内不再展示重复的“返回工作台”按钮。
- 验证三种模式按钮展示辅助说明。
- 验证结果空状态展示输出维度预告。
- 保留截图粘贴上传测试。
