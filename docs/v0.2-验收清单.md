# v0.2 验收清单

> 版本：v0.2  
> 对应里程碑：M1 产品闭环修复（P0-01 ~ P0-10）  
> 验收日期：待执行

## 验收环境要求

- Docker Compose 或手动启动：MySQL 8 + 后端 Go + 前端 Nginx
- 无 `AI_API_KEY` 配置时后端可正常启动（Mock Mode）
- 前端访问地址：`http://localhost:5173`
- 后端健康检查地址：`http://localhost:8080/api/health`

## 功能验收

### 1. Mock Mode

- [ ] 无 `AI_API_KEY` 环境变量时，后端启动成功
- [ ] `GET /api/system/status` 返回 `mock_mode: true`
- [ ] Mock 模式下运行五个工具均不产生外部 HTTP 请求
- [ ] Mock 模式输出可被前端解析和展示
- [ ] Settings 页显示「演示模式」标识
- [ ] Dashboard 侧边栏显示「演示」模式指示器

### 2. 报告列表

- [ ] 报告列表页展示标题、工具类型、状态、分数、摘要
- [ ] 按工具类型筛选可用（下拉选择）
- [ ] 按状态筛选可用（processing/succeeded/fallback/failed）
- [ ] 按时间/评分排序可用
- [ ] 分页可用（上一页/下一页）
- [ ] 筛选参数同步到 URL Query，刷新可恢复
- [ ] 无分数工具显示「无需评分」
- [ ] 加载中/错误/空列表/空筛选结果四种状态独立展示

### 3. 报告详情

- [ ] 详情页首屏展示：结论摘要 → 分数/等级 → 最严重 3 个问题 → Prompt → 下载
- [ ] 成功报告展示完整评分、问题、建议、Prompt、生成文件
- [ ] 降级（fallback）报告明确提示使用了降级数据
- [ ] 失败报告展示失败原因
- [ ] 处理中报告提示稍后刷新
- [ ] Codex Prompt 可一键复制，复制成功有视觉反馈
- [ ] 生成文件可下载
- [ ] 报告不存在时展示「报告未找到」并提供返回入口

### 4. 报告操作

- [ ] Markdown 报告可下载
- [ ] 删除前弹出确认对话框
- [ ] 确认对话框明确说明关联上传文件和生成文件会一起删除
- [ ] 删除成功后跳转到报告列表
- [ ] 取消删除不产生请求

### 5. Docker 部署

- [ ] `docker compose config` 无错误
- [ ] `docker compose up -d --build` 成功启动
- [ ] `http://localhost:5173` 可访问前端
- [ ] `http://localhost:8080/api/health` 返回健康状态
- [ ] 直接刷新 `/reports/{id}` 不返回 404（SPA fallback）

### 6. 事务一致性

- [ ] 生成文件写入失败时报告状态不会变为 succeeded
- [ ] 报告状态和生成文件状态保持一致

## 测试验收

- [ ] `cd backend && go test ./...` 全部通过
- [ ] `cd frontend && npm run test:unit` 全部通过
- [ ] `cd frontend && npm run build` 构建成功
- [ ] `cd frontend && npm run test:e2e` E2E 测试通过

## 安全验收

- [ ] ZIP Slip 路径穿越防护有效
- [ ] Markdown 渲染无明显 XSS 风险
- [ ] 下载文件名安全清洗
- [ ] 非法筛选参数不导致 SQL 注入或 500
- [ ] Prompt Injection 文本不覆盖系统指令

## 验证命令

```bash
# 后端测试
cd backend
go test ./...

# 前端构建
cd frontend
npm run build

# Docker 启动
docker compose up -d --build

# 健康检查
curl http://localhost:8080/api/health
curl http://localhost:5173

# 系统状态（查看 mock_mode）
curl http://localhost:8080/api/system/status
```
